# 🤖 QA AI Testing Tool

A complete web-based automated QA platform that parses requirement documents,
generates test cases using rule-based logic, executes Playwright tests, and
produces a professional HTML report — with zero paid AI APIs required.

---

## 📁 Project Structure

```
qa-tool/
├── backend/                    ← Node.js + Express API server
│   ├── server.js               ← Express entry point
│   ├── routes/testRoutes.js    ← API routes (execute, status, download)
│   ├── services/
│   │   ├── parser.js           ← Extract text from PDF/TXT/MD
│   │   ├── testGenerator.js    ← Rule-based test case generation
```

## ⚡ Quick Start

### Option A — One-Click Setup (Windows)
```
Double-click: setup.bat
```

### Option B — Manual Setup

**Step 1: Install Backend**
```bash
cd qa-tool/backend
npm install
```

**Step 2: Install Playwright**
```bash
cd qa-tool/playwright-tests
npm install
npx playwright install chromium
```

**Step 3: Install Frontend**
```bash
cd qa-tool/frontend
npm install
```

## 🚀 Running the Application

Open **two terminal windows**:

**Terminal 1 — Backend (port 5000)**
```bash
cd qa-tool/backend
npm start
```

**Terminal 2 — Frontend (port 3001)**
```bash
cd qa-tool/frontend
npm run dev
```

## 🎯 How to Use

1. **Enter Application URL** — The URL of the app to test (e.g., `http://localhost:3000`)
2. **Upload Requirement Document** — PDF, TXT, or MD file
3. **Click "Execute Testing"** — Watch the pipeline run in real time
4. **View Results** — See Pass/Fail per test case with execution time
5. **Download**:
   - **HTML Report** — Professional report, print to PDF
   - **Test Cases JSON** — Updated with execution status

## 🧠 How Test Generation Works

The system detects keywords in your requirement document and maps them to test cases:

| Keyword Detected | Module Generated | Tests |
|---|---|---|
| login, sign in, authentication | Authentication | Valid login, Wrong password, Empty fields |
| register, signup, create account | User Registration | Valid register, Duplicate email, Empty fields |
| payment, checkout, billing | Payment & Checkout | Valid card, Expired card, Empty fields |
| form, input, validation | Form Validation | Valid submit, Invalid email, Empty form |
| search, filter, find | Search & Filter | Valid search, No results |
| logout, sign out, session | Session Management | Logout success, Back button guard |
| password, forgot, reset | Password Management | Forgot flow, Invalid email |
| dashboard, overview, summary | Dashboard | Page load, Performance check |
| navigation, menu, nav | Navigation | Links work, Base URL loads |
| profile, account, settings | User Profile | Load profile, Auth guard |

**Always included:** Navigation baseline tests (app loads, links work)

## 🎭 Playwright Test Strategy

Generated tests use **adaptive selectors** — they try multiple CSS selectors to find elements:

```
Email input → tries: input[type="email"], input[name="email"], input[name="username"]...
Password    → tries: input[type="password"], input[name="password"]...
Submit btn  → tries: button[type="submit"], button:has-text("Login"), button:has-text("Sign In")...
```

Tests pass if the page **responds without crashing** — designed for any app's DOM structure.

## 🔧 Configuration

**Performance tuning** (`playwright-tests/playwright.config.js`):
```js
workers: 3,      // Parallel execution
timeout: 25000,  // Per-test timeout (ms)
retries: 0,      // No retries for speed
```

**Backend port** — set `PORT` env var (default: 5000)

## 🐛 Troubleshooting

**"pdf-parse module not found"**
```bash
cd backend && npm install pdf-parse
```

**"playwright not found"**
```bash
cd playwright-tests && npm install && npx playwright install chromium
```

**Port already in use**
- Backend: change `PORT=5001` in backend terminal
- Frontend: change port in `vite.config.js`

**Tests all show "Skipped"**
- This means Playwright ran but couldn't map results — check backend logs
- Try running: `cd playwright-tests && npx playwright test` manually
